# 🤖 JARVIS Mini — Local RAG Assistant

A fully offline, privacy-first Retrieval-Augmented Generation (RAG) assistant that runs entirely on your machine. Upload documents, ask questions, and get accurate answers powered by local LLMs — no cloud, no API keys, no data leaving your computer.

Built as a TFM project to explore the intersection of document intelligence, vector search, and local language models.

## ✨ Features

- **📄 Smart Document Ingestion** — Upload PDFs, DOCX, TXT or Markdown files. Tables and structured layouts are preserved using visual-aware extraction (`pdfplumber` with layout mode + regex normalization).
- **🔍 Semantic Search** — Documents are chunked respecting paragraph boundaries, embedded with `all-MiniLM-L6-v2`, and indexed in FAISS for sub-second retrieval.
- **🧠 Local LLM Generation** — Answers are generated by a quantized model running locally via `llama.cpp` (supports Qwen, Llama, and any GGUF model).
- **🌐 Web Fallback with Real Scraping** — When no relevant document is found, the system searches DuckDuckGo, scrapes the actual page content with `BeautifulSoup`, and feeds it to the LLM. Strict anti-hallucination prompts prevent the model from inventing real-time data.
- **⚡ Streaming Responses** — Tokens are streamed in real-time via SSE for a responsive chat experience.
- **🎨 Reflex UI** — A clean, modern chat interface built with Reflex (Python-native frontend framework).
- **🔒 100% Offline by Default** — Web search is optional and configurable. Everything else runs without internet.

## 🚀 Quick Start

### Prerequisites
- Python 3.10+
- ~2 GB disk space for the LLM model

### 1. Clone & Run

```powershell
git clone https://github.com/your-user/local-rag.git
cd local-rag
.\scripts\start-dev.bat
```

The script will automatically create a virtual environment, install all dependencies, and launch both the backend (`http://localhost:8000`) and the frontend (`http://localhost:3000`).

### 2. Manual Setup

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1

pip install -r backend\requirements.txt
pip install -r frontend\requirements.txt

# Terminal 1: Backend
python -m uvicorn backend.app.main:app --reload

# Terminal 2: Frontend
cd frontend
reflex run
```

### 3. Download a Model

Place a GGUF model file in `data/models/`. The default config expects `Qwen3.5-0.8B-BF16.gguf`. You can change the model path in `backend/app/core/config.py`.

## ⚙️ Configuration

All settings are centralized in [`backend/app/core/config.py`](backend/app/core/config.py):

| Setting | Default | Description |
|---|---|---|
| `embedding_model` | `all-MiniLM-L6-v2` | Sentence Transformer model for embeddings |
| `chunk_size` | `500` | Target chunk size (in words, ×5 for chars) |
| `chunk_overlap` | `50` | Overlap between chunks |
| `top_k` | `5` | Number of results to retrieve from FAISS |
| `relevance_threshold` | `0.45` | Minimum cosine similarity to consider a match |
| `llm_context_window` | `4096` | Context window of the local LLM |
| `llm_max_tokens` | `512` | Max tokens to generate per response |
| `enable_web_fallback` | `true` | Enable DuckDuckGo + scraping when no docs match |
| `web_search_max_results` | `3` | Number of web pages to scrape |

| Method | Endpoint | Description |

## 🧩 How It Works

1. **Ingestion**: Documents are parsed (PDF tables preserved via layout-aware extraction), split into chunks respecting paragraph boundaries, embedded, and stored in a FAISS index.
2. **Query**: The user's question is embedded and compared against the index. If a match exceeds the relevance threshold, the matching chunks are sent as context to the LLM.
3. **Web Fallback**: If no document matches, the system searches the web, scrapes the top results for real content (not just snippets), and uses that as context. Anti-hallucination prompts prevent the LLM from fabricating real-time data it doesn't have.
4. **Generation**: The local LLM generates a response using the provided context, streamed token-by-token to the UI.

## 🔧 Troubleshooting

### `llama-cpp-python` fails to install on Windows

If you don't have C++ build tools, install the prebuilt wheel:

```powershell
.\.venv\Scripts\pip install https://github.com/abetlen/llama-cpp-python/releases/download/v0.3.19/llama_cpp_python-0.3.19-cp312-cp312-win_amd64.whl
```

### DuckDuckGo returns empty results

DuckDuckGo may temporarily block automated requests (rate limiting). The system handles this gracefully by informing the user that the web search failed instead of hallucinating an answer. Wait a few minutes and try again.
